# Exercise 2: Agent Builder

## Overview

Today we'll build an **Agentforce Employee Agent** — an autonomous, employee-facing
agent designed to support PSD officers. Its mission is to handle the end-to-end
"administrative tail" of complex citizen cases: it doesn't just draft emails, it reasons
through policy, updates records, and proactively flags gaps to human officers.

You will:

- **2A — Create a New Agent with Default Topics:** stand up an agent equipped with
  standard out-of-the-box actions and test it in the Employee Console.
- **2B — Enhance with Topics & Actions:** define a custom **Case Follow-up** topic and
  link your **Citizen Service FollowUp** prompt as an Action — watching the reasoning
  engine decide when to act.

::: tip Topics vs. Actions
Think of a **Topic** as a department (e.g. "Policy Specialist") and an **Action** as a
specific task that department can perform (e.g. "Calculate Grant Eligibility").
:::

Continue to **2A: New Agent with Default Topics** below.
